Q189842: XCLN: Client Folder Views May Prevent Any Items From Appearing

Article: Q189842
Product(s): Microsoft Exchange
Version(s): MACINTOSH:8.0; WINDOWS:4.0,5.0,8.0; :8.0,8.01,8.02,8.03
Operating System(s): 
Keyword(s): 
Last Modified: 06-DEC-2000

-------------------------------------------------------------------------------
The information in this article applies to:

- Microsoft Exchange Windows 3.x client, versions 4.0, 5.0 
- Microsoft Exchange Windows 95/98 client, versions 4.0, 5.0 
- Microsoft Exchange Windows NT client, versions 4.0, 5.0 
- Microsoft Exchange Macintosh client, versions 4.0, 5.0 
- Microsoft Outlook for Macintosh, Exchange Server Edition, version 8.0 
- Microsoft Outlook, Exchange Server Edition, version 8.0, used with:
   - the operating system: Microsoft Windows versions 3.1, 3.11 
- Microsoft Outlook 97, versions 8.0, 8.01, 8.02, 8.03 
- Microsoft Outlook 98 
-------------------------------------------------------------------------------

SYMPTOMS
========

When you use an Outlook or Exchange client to access your mailbox, checking the
Sent Items folder may show nothing even though you have selected "Save Copies of
Messages in Sent Items folder" in the Options menu.

CAUSE
=====

This can occur if the default views have changed and are filtering for messages
that do not meet any of the selected criteria.

WORKAROUND
==========

Set the folder views back to the default settings or modify the current folder
views to allow characteristics that will allow messages to be seen. This
situation can occur with any folders in your mailbox if the filters for views
are set incorrectly. For instance, selecting messages "Flagged for Next Seven
Days" in the View menu may display nothing in the folder if there are no
messages that meet this criteria.
