Q190774: FIX: Keyboard Can Cause Component Gallery to Close

Article: Q190774
Product(s): Microsoft FoxPro
Version(s): WINDOWS:6.0
Operating System(s): 
Keyword(s): kbAppSetup kbvfp600 kbvfp600bug kbXBase kbVS600sp3fix kbGrpDSFox kbTools
Last Modified: 10-JUL-1999

-------------------------------------------------------------------------------
The information in this article applies to:

- Microsoft Visual FoxPro for Windows, version 6.0 
-------------------------------------------------------------------------------

SYMPTOMS
========

After you use the Component Gallery for some time, pressing keys on the keyboard
can cause the Component Gallery to close unexpectedly.

RESOLUTION
==========

This bug has been fixed in the updated version of the Setup Wizard Vfp6swiz.exe
dated 10/08/1998.

STATUS
======

Microsoft has confirmed this to be a problem in the Microsoft products that are
listed at the beginning of this article.This bug was corrected in the Setup
Wizard Vfp6swiz.exe dated 10/08/1998.

This bug was corrected in Visual Studio 6.0 Service Pack 3.

For more information about Visual Studio service packs, please see the following
articles in the Microsoft Knowledge Base:

  Q194022 INFO: Visual Studio 6.0 Service Packs, What, Where, Why

  Q194295 HOWTO: Tell That Visual Studio 6.0 Service Packs Are Installed

MORE INFORMATION
================

Steps to Reproduce Behavior
---------------------------

1. Start the Component Gallery.

2. Click on the Foundation Classes folder.

3. Press any key on the keyboard and note that the Component Gallery closes
  unexpectedly.

REFERENCES
==========

For more information on the updated Setup Wizard, please see the following World
Wide Web URL:

http://msdn.microsoft.com/vfoxpro/downloads/updates.asp
