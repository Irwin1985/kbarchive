Q165302: WD97: Numbered Page Break Added When File Opened in 7.0

Article: Q165302
Product(s): Word 97 for Windows
Version(s): WINDOWS:97
Operating System(s): 
Keyword(s): 
Last Modified: 14-NOV-2000

-------------------------------------------------------------------------------
The information in this article applies to:

- Microsoft Word 97 for Windows 
-------------------------------------------------------------------------------

SYMPTOMS
========

If you have Outline Numbering applied to a Word 97 document and you format page
breaks, section breaks, or column breaks as a heading style, then when you open
the file in Word 7.0, Heading Numbering may be incorrectly applied to the breaks
instead of the text near the breaks.

CAUSE
=====

This is a limitation in the Word 7.0 heading numbering feature.

WORKAROUND
==========

Format the break with a different style, such as Normal.

-or-

Obtain the Word 97-2000 Import Converter for Word 6.0 and Word 7.0. For
information about how to obtain the import converter, please see the following
article in the Microsoft Knowledge Base:

  

  Q162214 WD: How to Obtain the Word 97-2000 Import Converter

MORE INFORMATION
================

In Word 7.0, there are some situations where a break formatted with a heading
style takes on heading numbering. This problem was corrected in Word 97;
however, when you save as Word 6.0 from Word 97 and then open the file in Word
7.0, the heading numbering may be incorrectly applied to the breaks.

REFERENCES
==========

For more information about exchanging documents between Word 97 and earlier
versions of Word, please see the following articles in the Microsoft Knowledge
Base:

  

  Q162602 WD97: Questions About Saving in the Word 6.0/95 File Format

  Q162987 WD: Supported Non-Word File Formats

  Q157091 WD97: How to Open Documents into an Earlier Version of Word
