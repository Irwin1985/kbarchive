Q197764: WD97: Table Headings Do Not Appear in Normal View

Article: Q197764
Product(s): Word 97 for Windows
Version(s): WINDOWS:97
Operating System(s): 
Keyword(s): word97 kbtable
Last Modified: 14-NOV-2000

-------------------------------------------------------------------------------
The information in this article applies to:

- Microsoft Word 97 for Windows 
-------------------------------------------------------------------------------


SYMPTOMS
========

When you add headings to a table by clicking Headings on the Table menu, Word
does not display the headings at the top of each page on which the table appears
in normal view. Word only displays the headings at the top of the table.

In contrast, Word displays the table headings at the top of each page on which
the table appears in page layout view or print preview.

CAUSE
=====

When you use the Headings command, Word treats the table heading rows like
headers instead of table rows and, as with normal headers, Word does not display
them in normal view.
