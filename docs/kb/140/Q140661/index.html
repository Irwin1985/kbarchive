Q140661: Encarta 97 Deluxe: CD swapping problem with NetWare/Win 3.x

Article: Q140661
Product(s): Microsoft Home Multimedia Titles
Version(s): 
Operating System(s): 
Keyword(s): 
Last Modified: 21-DEC-2001

-------------------------------------------------------------------------------
The information in this article applies to:

- Microsoft Encarta 96 Encyclopedia for Windows 
- Microsoft Windows for Workgroups 
-------------------------------------------------------------------------------

SYMPTOMS
========

You have problems swapping compact discs when you use the deluxe edition of
Encarta with Windows 3.x and Novell NetWare.

The problem appears when you are asked to insert a compact disc (disc 1 or 2).
After you insert the disc, Encarta doesn't recognize that the appropriate disc
is in the CD-ROM drive.

RESOLUTION
==========

If you encounter this problem, you need to download a patch that upgrades your
copy of the Encarta. The patch allows Encarta Encyclopedia to recognize the
correct compact disc after a disc swap.

Download the patch only if all the following conditions are true:

- You're using Windows 3.1 or 3.11

  -and-

- You're using Novell NetWare client software

  -and-

- You're using Encarta 97 Encyclopedia Deluxe Edition, and

  -and-

- You're having difficulties swapping compact discs.

Downloading the Patch
---------------------

The self-extracting E97nwfix.exe file is available as a free download from the
Microsoft Download Center.

The following file is available for download from the Microsoft Download Center:

  E97nwfix.exe
  (http://download.microsoft.com/download/encarta97win/Update/1/WIN98/EN-US/E97nwfix.exe)

For additional information about how to download Microsoft Support files, click
the article number below to view the article in the Microsoft Knowledge Base:

  Q119591 How to Obtain Microsoft Support Files from Online Services

Microsoft used the most current virus detection software available on the date of
posting to scan this file for viruses. Once posted, the file is housed on secure
servers that prevent any unauthorized changes to the file.

NOTE: This conflict occurs only under Windows 3.1 or 3.11. No conflict occurs
under Windows 95 or Windows NT.
