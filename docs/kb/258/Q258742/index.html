Q258742: Intermittent "Access Denied" Error Messages While Printing

Article: Q258742
Product(s): Microsoft Windows NT
Version(s): winnt:4.0
Operating System(s): 
Keyword(s): kbWinNT400PreSP7Fix
Last Modified: 08-MAY-2002

-------------------------------------------------------------------------------
The information in this article applies to:

- Microsoft Windows NT Server version 4.0 
- Microsoft Windows NT Workstation version 4.0 
-------------------------------------------------------------------------------

SYMPTOMS
========

When you are printing or administering printers on remote servers, you may
receive intermittent "Access denied" error messages.

RESOLUTION
==========

A supported fix is now available from Microsoft, but it is only intended to
correct the problem that is described in this article. Apply it only to
computers that are experiencing this specific problem.

To resolve this problem, contact Microsoft Product Support Services to obtain the
fix. For a complete list of Microsoft Product Support Services phone numbers and
information about support costs, visit the following Microsoft Web site:

  http://support.microsoft.com/default.aspx?scid=fh;EN-US;CNTACTMS

NOTE: In special cases, charges that are ordinarily incurred for support calls
may be canceled if a Microsoft Support Professional determines that a specific
update will resolve your problem. The usual support costs will apply to
additional support questions and issues that do not qualify for the specific
update in question.

The English version of this fix should have the following file attributes or
later:

  Date        Time    Size     File name     Platform
  ---------------------------------------------------
  29/03/2000  02:35   342,800  Rpcrt4.dll    Intel
  29/03/2000  02:35    80,144  Win32spl.dll  Intel
  29/03/2000  02:34   579,344  Rpcrt4.dll    Alpha
  29/03/2000  02:34   136,464  Win32spl.dll  Alpha



WORKAROUND
==========

There are several workarounds. You can stop and start the spooler on the client
computer, you can print to another printer, or you can try waiting and repeating
the same operation later.

STATUS
======

Microsoft has confirmed that this is a problem in the Microsoft products that
are listed at the beginning of this article.
