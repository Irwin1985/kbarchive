Q194601: WD97:Bottom Border Deleted in Converted WordPerfect 5.x Document

Article: Q194601
Product(s): Word 97 for Windows
Version(s): WINDOWS:97
Operating System(s): 
Keyword(s): kbdta kbconversion winword word97 kbtable
Last Modified: 14-NOV-2000

-------------------------------------------------------------------------------
The information in this article applies to:

- Microsoft Word 97 for Windows 
-------------------------------------------------------------------------------

SYMPTOMS
========

In Word 97, when you open a WordPerfect 5.x for MS-DOS or Windows document that
contains a table with borders, the bottom cell border for the last row on each
page of a multipage table does not appear in page layout view and is not
printed.

WORKAROUND
==========

After conversion, reapply the border to the cells at the bottom of the page. To
do this, either select the affected cell or the entire table, click Borders And
Shading on the Format menu, and reapply the appropriate border.

STATUS
======

Microsoft has confirmed this to be a problem in the Microsoft products that are
listed at the beginning of this article.
