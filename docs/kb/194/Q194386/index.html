Q194386: X400: Can’t Send Messages to MTA with Checkpoint Size Set to 10

Article: Q194386
Product(s): Microsoft Mail For PC Networks
Version(s): MS-DOS:3.2
Operating System(s): 
Keyword(s): 
Last Modified: 20-MAY-2002

-------------------------------------------------------------------------------
The information in this article applies to:

- Microsoft Mail Gateway to X.400, version 3.2 
-------------------------------------------------------------------------------


SYMPTOMS
========

If you attempt to send a message through the Microsoft Mail Gateway to X.400 to
a foreign system on which the message transfer agent (MTA) is configured for a
checkpoint size of 10, the message may not be sent successfully, and the
following error message may be displayed:

  DEBUG (T4_SEND): CONNECTION 32: TP_SEND() FAILED: 3

If you attempt to send another message through the Microsoft Mail Gateway to
X.400 after the above error message is displayed, you are unable to do so. Note
that this problem only occurs if you attempt to send the message using Transport
Class 4 (TP4). The problem does not occur if you send the message using
Transport Class 0 (TP0).

RESOLUTION
==========

A supported fix that corrects this problem is now available from Microsoft, but
has not been fully regression-tested and should be applied only to systems
experiencing this specific problem.

To resolve this problem, contact Microsoft Product Support Services to obtain the
fix. For a complete list of Microsoft Product Support Services phone numbers and
information on support costs, please go to the following address on the World
Wide Web:

  http://support.microsoft.com/default.aspx?scid=fh;EN-US;CNTACTMS

The English version of this fix should have the following file attributes or
later:

  File Name      Version
  ----------------------
  X400gate.exe   3.2.11

This hotfix has been posted to the following Internet location as X400gy2k.exe:

  ftp://ftp.microsoft.com/bussys/mail/gateways-public/All-Y2K/


STATUS
======

Microsoft has confirmed this to be a problem in Microsoft Mail Gateway to X.400
version 3.2.
