Q289848: Profiles Are Not Saved To the Roaming Profile Location on NT 4.0

Article: Q289848
Product(s): Microsoft Windows NT
Version(s): 4.0 SP4,4.0 SP5
Operating System(s): 
Keyword(s): kbWinME
Last Modified: 08-AUG-2001

-------------------------------------------------------------------------------
The information in this article applies to:

- Microsoft Windows NT Workstation versions 4.0 SP4, 4.0 SP5 
-------------------------------------------------------------------------------

SYMPTOMS
========

When you are logged on to a computer that is running Windows NT Workstation 4.0
Service Pack 4 or Service Pack 5, profiles may not be saved to the location
specified for roaming profiles.

CAUSE
=====

This behavior can occur if both Rockwell's DeskSet and Norton AntiVirus are
installed on the computer, causing the profile to lock.

RESOLUTION
==========

To resolve this behavior, contact Symantec or Rockwell.

WORKAROUND
==========

To work around this behavior, remove either Norton AntiVirus or DeskSet.
