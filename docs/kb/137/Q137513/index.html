Q137513: User Manager Cannot Rename a Group

Article: Q137513
Product(s): Microsoft Windows NT
Version(s): winnt:3.51,4.0
Operating System(s): 
Keyword(s): kbOSWinNT351 kbOSWinNT400
Last Modified: 08-AUG-2001

-------------------------------------------------------------------------------
The information in this article applies to:

- Microsoft Windows NT Workstation versions 3.51, 4.0 
- Microsoft Windows NT Server versions 3.51, 4.0 
-------------------------------------------------------------------------------

SYMPTOMS
========

In User Manager (or User Manager for Domains), it is not possible to rename a
Group. If you select a Group, and choose the User menu, the Rename command is
unavailable.

RESOLUTION
==========

To workaround this problem, copy the group to a different name and then delete
the original group.

STATUS
======

Microsoft has confirmed this to be a problem in Windows NT versions 3.51 and
4.0. We are researching this problem and will post new information here in the
Microsoft Knowledge Base as it becomes available.


Additional query words: usrmgr.exe musrmgr.exe
