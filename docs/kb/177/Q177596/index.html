Q177596: MarkVision TCP/IP Auto-Discovery for Windows 95 and Windows NT

Article: Q177596
Product(s): Windows for Workgroups and Windows NT Networking Issues
Version(s): WinNT:4.0;Windows:95
Operating System(s): 
Keyword(s): kbprint
Last Modified: 07-SEP-2001

-------------------------------------------------------------------------------
The information in this article applies to:

- Microsoft Windows NT Workstation version 4.0 
- Microsoft Windows NT Server version 4.0 
- Microsoft Windows 95 
-------------------------------------------------------------------------------

SUMMARY
=======

Lexmark now provides a TCP/IP auto-discovery capability in its utilities,
MarkVision for Windows 95, and MarkVision for Windows NT 4.0.

TCP/IP auto-discovery allows you to:

- Discover a specific TCP/IP printer on your entire network.

- Discover TCP/IP printers on a specific subnet of your network.

- Discover TCP/IP printers on a given TCP/IP printer's subnet.

- Choose from the discovered list of printers, those to be monitored by
  MarkVision.

- Save the list of discovered printers to a file to be imported into MarkVision
  running on another computer.

- Integrate into DNS/WINS.

The TCP/IP Auto-Discovery function runs on both Windows 95 and Windows NT 4.0.
Microsoft's TCP/IP protocol stack (standard on both Windows 95 and Windows NT
4.0) is required. You must install MarkVision for Windows 95 Version 3.03, or
MarkVision for Windows NT 4.0 Version 3.03, before you install the upgrade.

MORE INFORMATION
================

TCP/IP Auto-Discovery is currently available in the form of an upgrade on the
World-Wide Web at http://www.lexmark.com/ from the Printer Software tab on the
MarkVision icon.

For further technical assistance you may contact the Lexmark Technical Support
Center at (606) 232-3000.
