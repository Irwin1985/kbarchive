Q146048: XCLN: Changing Fonts for Richedit Field Doesn’t Resize Field

Article: Q146048
Product(s): Microsoft Exchange
Version(s): 4.0 5.0
Operating System(s): 
Keyword(s): kbusage
Last Modified: 16-MAR-1999

-------------------------------------------------------------------------------
The information in this article applies to:

- Microsoft Exchange Windows 3.x client, versions 4.0, 5.0 
- Microsoft Exchange Windows NT client, versions 4.0, 5.0 
- Microsoft Exchange Windows 95/98 client, versions 4.0, 5.0 
-------------------------------------------------------------------------------

SYMPTOMS
========

When you drop a new rich entry field onto a newly created forms canvas and
change the font size property of that rich entry field, when you reposition this
field, the horizontal size of the field does not correctly resize to reflect the
new font size. The height property of the field does correctly show the new size
related to the new font size.
