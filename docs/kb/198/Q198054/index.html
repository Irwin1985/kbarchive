Q198054: WD97: Known Issues Using "Windows 95 for Pen Computing"

Article: Q198054
Product(s): Word 97 for Windows
Version(s): WINDOWS:97
Operating System(s): 
Keyword(s): word97
Last Modified: 14-NOV-2000

-------------------------------------------------------------------------------
The information in this article applies to:

- Microsoft Word 97 for Windows 
-------------------------------------------------------------------------------



SYMPTOMS
========

This article lists known problems using Windows 95 for Pen Computing and Word
for Windows.

The following gestures do not work:

- Edit Text gesture (check mark circled)

- Insert Text gesture (caret circled)

- Context Menu gesture (M circled), can't right-click for Context menus

- Select gesture (lasso tap)

The following symptoms and problems may be experienced:

- Pen Windows display does not refresh correctly when the pen is near the
  digitizer pad.

- When the pen is near (or in contact with the digitizer) and the barrel button
  is pressed, the cursor changes to an I-Beam.

- The pen "drag handle" is not shown. This is a small button that floats along
  with the insertion point, allowing access to a context menu and easier access
  with a pen.

- You cannot drag a selection with the pen.

- When you neatly print characters using the pen stylus, the characters are
  connected as if you had not lifted the pen between printed characters.

WORKAROUND
==========

Context Menu Gesture (M circled)
--------------------------------

Press SHIFT+F10 with the pen at the point where you want a context menu.

Display Refresh Symptom
-----------------------

Keep the pen away from the digitizer pad when not in use. For additional
information, please see the following article in the Microsoft Knowledge Base:

  Q110398 Drag-and-Drop Editing Not Functional with Pen Windows

STATUS
======

Microsoft has confirmed this to be a problem in the Microsoft products listed at
the beginning of this article.
